# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What This Project Is

**jr-toolkit** is a Spec-Driven Development (SDD) toolkit that installs Claude.ai skills via `npx`. It covers the full development lifecycle: from raw idea → vision → architecture → roadmap → spec → implementation → verification → bug fixing.

## CLI Commands

```bash
# Install/manage skills locally
npx jr-toolkit install
npx jr-toolkit uninstall
npx jr-toolkit list
npx jr-toolkit help

# Manual installation (bash)
./install-all.sh
```

There is no build step, no transpilation, and no test suite yet (`npm test` is a placeholder).

## Architecture

### Entry Point

`bin/jr-toolkit.js` — Node.js CLI that copies skill files from the package into `~/.claude/skills/` and `~/.claude/commands/`.

### Skills (in `skills/`)

Each skill directory contains:
- `SKILL.md` — the actual Claude.ai skill instructions (500–1000 lines; this is the "source of truth" for behavior)
- `command.md` — user-facing slash command help shown in Claude.ai
- `install.sh` — copies files to `~/.claude/`

**Project kickoff (new projects):**

| Skill | Slash Command | Input → Output |
|---|---|---|
| `jr-vision` | `/jr-vision` | Raw idea → `docs/vision.md` |
| `jr-arch` | `/jr-arch` | `docs/vision.md` → `docs/architecture.md` + `PROJECT.md` |
| `jr-roadmap` | `/jr-roadmap` | Vision + Architecture → `docs/roadmap.md` + spec placeholders |
| `jr-init` | `/jr-init` | Existing project → `PROJECT.md` |

**Feature development:**

| Skill | Slash Command | Input → Output |
|---|---|---|
| `jr-build-spec` | `/jr-build-spec` | Rough requirement → `specs/feature.md` (Draft) |
| `jr-iterate-spec` | `/jr-iterate-spec` | Existing spec + change → new version (Draft) |
| `jr-exe-spec` | `/jr-exe-spec` | Approved spec → code (Implemented) |
| `jr-verify-spec` | `/jr-verify-spec` | Implemented spec → coverage report (Verified) |

**Maintenance:**

| Skill | Slash Command | Input → Output |
|---|---|---|
| `jr-fix-spec` | `/jr-fix-spec` | Bug report `.md` → fix applied + hotfix entry in original spec |
| `jr-status` | `/jr-status` | — → dashboard of all specs |

### Spec Lifecycle

```
Pending → Draft → Implemented → Verified
             ↑                      |
             └──── jr-iterate ───────┘  (new version)

Bug lifecycle: specs/fixes/bug.md → jr-fix-spec → Resolved
```

Specs live in `specs/[feature].md`. Bug specs live in `specs/fixes/`. Project context is stored in `PROJECT.md`. Code generated by `jr-exe-spec` includes `// spec: specs/[name].md` traceability comments.

### Stack Patterns

`skills/jr-exe-spec/references/stack-patterns.md` — coding conventions for 20+ stacks (React, Next.js, Vue, Laravel, WordPress, Shopify, Tailwind, Docker, etc.) used by `jr-exe-spec` when generating code.
